# Task List

## Task 1: MVP-Kernfunktion definieren: Keyword-Analyse & Monitoring
**Category:** development
**Effort:** medium
**Deadline:** 5 days

Definiere die *eine* Kernfunktion für den MVP, die den grössten Mehrwert bietet und mit No-code umgesetzt werden kann. Fokussiere auf Keyword-Analyse und Ranking-Monitoring für eine einzelne Unternehmenswebsite und ein Google Business Profile. Skizziere den genauen Prozess und die Datenpunkte (z.B. primäres Keyword, aktuelle Position, Suchvolumen).

### Acceptance Criteria
- [ ] Implementation complete
- [ ] Tests written
- [ ] Documentation updated


## Task 2: API-Integration (MVP): Google Search Console & SERP API
**Category:** development
**Effort:** high
**Deadline:** 7 days

Erkunde und integriere die notwendigen APIs, um die im MVP definierte Kernfunktion (Keyword-Analyse & Monitoring) zu ermöglichen. Beginne mit der Google Search Console API für eigene Daten und einer SERP API (z.B. SerpApi, Apify) für öffentliche Ranking-Daten. Teste die Konnektivität mit n8n und speichere erste Testdaten in Supabase. Konzentriere dich auf die Extraktion der wichtigsten Metriken.

### Acceptance Criteria
- [ ] Implementation complete
- [ ] Tests written
- [ ] Documentation updated


## Task 3: Landing Page für erste User-Tests & Feedback
**Category:** marketing
**Effort:** low
**Deadline:** 3 days

Erstelle eine einfache Landing Page mit Fokus auf das definierte Problem und die MVP-Lösung. Integriere ein Anmeldeformular für Interessenten, die den MVP testen möchten. Sammle hierüber frühzeitig Feedback potenzieller Nutzer, noch bevor der MVP fertig ist. Die Seite muss nicht perfekt sein, sondern klar die Idee kommunizieren.

### Acceptance Criteria
- [ ] Implementation complete
- [ ] Tests written
- [ ] Documentation updated


## Task 4: MVP-Prototyp (UI/UX): Keyword-Dashboard
**Category:** development
**Effort:** medium
**Deadline:** 7 days

Entwickle einen klickbaren Prototyp in Lovable/Bubble, der das definierte Keyword-Analyse- und Monitoring-Feature darstellt. Der Prototyp soll zeigen, wie Nutzer Keywords eingeben, Positionen verfolgen und einfache Berichte sehen können. Fokus auf intuitive Bedienung und die Visualisierung der wichtigsten Datenpunkte. Eine funktionierende Backend-Anbindung ist im ersten Schritt noch nicht zwingend, aber eine klare Vorstellung davon, wie die Daten fliessen, ist wichtig.

- [ ] Implementation complete
- [ ] Tests written
- [ ] Documentation updated
